
# STIG: WN11-CC-000315 â€“ Always install with elevated privileges must be disabled

### Identifiers
- STIG ID: WN11-CC-000315
- Vuln ID: V-253260 (DISA STIG)

## System / Scope

- **System:** Microsoft Windows 11 Pro 64-bit (Azure virtual machine, x64), standalone
- **Environment:** Azure-hosted virtual machine used for STIG testing
- **Role of system:** User workstation endpoint used for security testing and STIG remediation within a controlled lab environment

## Finding

- Windows Installer was configured to allow elevated privilege installations.
- This configuration allowed potential privilege escalation.

## Security Impact

- Unauthorized software could be installed with administrative privileges.
- Increased risk of system compromise.
- Violation of least privilege principles.

## Remediation Performed

- Reviewed the existing Windows Installer configuration.
- Disabled elevated installation privileges via registry policy.

## PowerShell Remediation Script

```powershell
New-Item -Path 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\Installer' -Force | Out-Null

Set-ItemProperty `
  -Path 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\Installer' `
  -Name 'AlwaysInstallElevated' `
  -Value 0 `
  -Type DWord

```

Validation

Confirmed the registry value was correctly set.

Performed a Tenable compliance rescan using the Microsoft_Windows_11_STIG policy.

Verified that STIG WN11-CC-000315 passed successfully after remediation.
