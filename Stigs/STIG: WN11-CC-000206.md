# STIG: WN11-CC-000206 â€“ Delivery Optimization must not use Internet peering

### Identifiers
- STIG ID: WN11-CC-000206
- Vuln ID: V-253267 (DISA STIG)

## System / Scope

- **System:** Microsoft Windows 11 Pro 64-bit (Azure virtual machine, x64), standalone
- **Environment:** Azure-hosted virtual machine used for STIG testing
- **Role of system:** User workstation endpoint used for security testing and STIG remediation within a controlled lab environment

## Finding

- Delivery Optimization was configured to allow Internet-based peering.
- Update content could be shared with external peers.

## Security Impact

- Potential exposure of update data to untrusted systems.
- Increased attack surface.
- Violation of enterprise update control policies.

## Remediation Performed

- Reviewed Delivery Optimization configuration.
- Configured the system to use HTTP-only download mode.

## PowerShell Remediation Script

```powershell
New-Item -Path 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\DeliveryOptimization' -Force | Out-Null

Set-ItemProperty `
  -Path 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\DeliveryOptimization' `
  -Name 'DODownloadMode' `
  -Value 0 `
  -Type DWord

```

Validation

Confirmed the registry value was correctly set.

Performed a Tenable compliance rescan using the Microsoft_Windows_11_STIG policy.

Verified that STIG WN11-CC-000206 passed successfully after remediation.
